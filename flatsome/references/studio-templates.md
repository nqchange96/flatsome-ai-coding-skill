# Flatsome Studio — layout starting points (catalog)

Flatsome ships **UX Studio**: a library of ready-made, fully-native layouts you can import into
any page and then edit in UX Builder. Use it as a **starting point** to avoid blank-page output
and to keep pages varied (see the "vary the layout" convention in [SKILL.md](../SKILL.md)).

> **Provenance / freshness:** this is a curated **pointer index**, not embedded layout data.
> Template IDs and preview URLs are a **snapshot (2026-06-22)** of the official UX Themes demo
> site and may change — always confirm against the live Studio before relying on an ID. Category
> names reflect how the official Studio groups layouts; verify in your own install.

## How to use Studio

- **In UX Builder:** add a block → **UX Studio** → browse by category → **Import**. The imported
  layout is plain shortcodes (`section/row/col/...`), so you can restyle and re-arrange it freely.
- **Browse/preview online:** the official demo layouts live at
  `https://demos.uxthemes.com/layouts/?p=<TEMPLATE_ID>` — open an ID to see it before importing.
- **As an AI starting point:** when a brief matches a category below, pick a layout as the
  skeleton, then adapt content + apply the project's conventions (one block = one section, flat
  structure, unique classes, builder-first styling). Don't clone verbatim — vary it.
