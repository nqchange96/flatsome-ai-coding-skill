# UX Builder, Blocks, Header & Menus

## What UX Builder is

UX Builder is Flatsome's **live front-end page builder**. It edits the page visually but stores
the result as **shortcodes** in `post_content`. Open it via:
- Page/post edit screen → **"Edit with UX Builder"** button, or
- Frontend admin bar → **"Edit with UX Builder"**.

Because output is shortcodes, you can author layouts by hand (see
[shortcodes.md](shortcodes.md)) and paste them into the WordPress code editor.

### Builder element categories
- **Layout:** Section, Row, Column, Gap, Divider
- **Content:** Text, Title, Image, Banner, Image Box, Featured Box, Video, Gallery, Button,
  Icon, HTML
- **Shop:** Products, Product Categories, Add to Cart, and (in product layouts) the
  `ux_product_*` elements
- **Product Page** (only when editing a product layout): Gallery, Title, Price, Add to Cart,
  Tabs, Related, Up-sells, Product Hooks, etc.

### Get exact shortcode from the builder
1. Blank page → open UX Builder → build & style the element.
2. Save → open the page in the normal WP **code editor** → copy the generated shortcode.
Works anywhere: header, block, widget, another page.

## UX Blocks (reusable content)

**Blocks** is a custom post type (`Dashboard → Blocks`) for reusable layouts. Build a block in
UX Builder, then insert it anywhere:
- Shortcode: `[block id="123"]` (post ID) — works in pages, widgets, other blocks, theme code.
- Assign blocks to: **header positions**, **footer**, **mega menus**, **custom product/shop
  layouts**, **popups**, **404 page content**, etc.

**Naming conventions from the docs:**
- `CPL - ...` custom product layout (see [woocommerce.md](woocommerce.md))
- `CML - ...` custom menu layout / mega menu (below)

**Rule:** every block assigned to a menu must be **unique** — don't reuse one block in two
menu items.

## Header builder

Theme Options → **Header** uses a drag-and-drop element system across three rows
(Top bar / Main / Bottom), each with **Left / Center / Right** zones. Drag elements like:
`Logo`, `Menu`, `Search`, `Cart`, `Account`, `Button`, `Social`, `Text`, `Block`, `Divider`,
`Contact`.

- Insert custom content via the **Block** or **HTML/Text** header element.
- A shortcode generated by UX Builder can be pasted into a header **Text** element.
- Configure transparent/sticky header, mobile header layout, and **menu locations** here and in
  `Appearance → Menus`.

## Navigation & menus

Menu locations (`Appearance → Menus → Manage Locations`) include:
`Main Menu`, `Main Menu - Mobile`, `Top Bar Menu`, `Footer`, plus widget/sidebar menus.

### Mega Menu (custom dropdown)
Source: docs.uxthemes.com/article/391.
1. **Create a block:** `Dashboard → Blocks → Add new` (name it e.g. `CML - Mega 1`).
2. **Attach to a menu item:** `Appearance → Menus` → select/add the parent menu item →
   open the **Design** options → choose your block as the **Custom Menu Layout** → save.
3. **Edit:** on the frontend, hover the menu → **Edit Block** → build the dropdown in UX
   Builder. Each menu-assigned block must be unique.
- Requires Flatsome ≥ 3.13.0, WordPress ≥ 5.4.
- For mobile: child items show in the sidebar nav; you can configure a **separate sidebar
  (mobile) menu** if needed.

### Other menu features
- **Vertical menu**, **menu dropdown** styling, **mobile sidebar menu elements**
  (add search/account/custom HTML to the off-canvas menu), and **personal menu labels** are all
  configurable — see docs category 38 (Navigation).

## Pattern: dynamic / interactive components → register a PHP shortcode (NOT `[ux_html]<script>`)

For anything that needs **JavaScript interaction or data iteration** (custom sliders with
avatar-selectors, autoplay showcases, filtered lists, anything stateful), do **not** stuff
`<script>` / `<style>` / `<link>` into `[ux_html]` — **UX Builder can strip them on save**.
Instead register a shortcode in the **child theme `functions.php`** (or a Code Snippets plugin)
and place the clean token `[my_component]` in the layout.

- **Why it survives:** UX Builder only ever sees the token `[my_component]` — it never touches
  the HTML/JS the shortcode prints, so your script is 100% safe and the component is reusable.
- **Real example:** a testimonial showcase (large image + quote + avatar selector + autoplay)
  registered as `[dk_testimonials]`, reading attachment IDs via `wp_get_attachment_image_url()`.

```php
// child theme functions.php
add_shortcode('my_component', function ($atts) {
    $a = shortcode_atts(['ids' => ''], $atts);
    $ids = array_filter(array_map('trim', explode(',', $a['ids'])));

    ob_start(); ?>
    <div class="my-component" data-autoplay="1">
      <?php foreach ($ids as $id):
        $url = wp_get_attachment_image_url((int) $id, 'large'); ?>
        <figure class="my-component__slide"><img src="<?php echo esc_url($url); ?>" alt=""></figure>
      <?php endforeach; ?>
    </div>
    <script>/* component JS — safe here, UX Builder never sees it */</script>
    <?php
    return ob_get_clean();
});
```
Then in the page/builder: `[my_component ids="123,124,125"]`. Put the component's CSS in Theme
Options → Custom CSS (or enqueue it). **Rule:** interactive/data-driven → PHP shortcode; reserve
`[ux_html]` for static embeds (iframe / map / third-party widget).

## Rendered HTML structure of theme regions (Flatsome standard)

Header/menu/footer/sidebar are **theme template output** (not page shortcodes). These are the
standard classes Flatsome renders — use them as CSS targets. (Verified against live Flatsome
sites; class names are theme-standard, not project-specific.)

### Header — 3 rows, each with left/center/right zones
```
#header.header > div.header-wrapper
  ├─ #top-bar.header-top            (Top bar row; often hide-for-medium / nav-dark)
  │    └─ div.flex-row.container
  │         ├─ div.flex-col.flex-left   > ul.nav.nav-left ...
  │         ├─ div.flex-col.flex-center > ul.nav.nav-center ...
  │         └─ div.flex-col.flex-right  > ul.nav.nav-right ...
  ├─ #masthead.header-main          (Main row)
  │    └─ div.header-inner.flex-row.container.logo-left
  │         ├─ #logo.flex-col.logo > a
  │         ├─ div.flex-col.hide-for-medium.flex-left  > ul.header-nav.header-nav-main.nav.nav-left
  │         ├─ div.flex-col.hide-for-medium.flex-right > ul.header-nav.header-nav-main.nav.nav-right
  │         └─ div.flex-col.show-for-medium ...        > ul.mobile-nav.nav   (mobile toggle)
  └─ #wide-nav.header-bottom.wide-nav  (Bottom row; hide-for-medium)
       └─ div.flex-row.container > div.flex-col ... > ul.nav
```
- `.flex-row.container` = each row; `.flex-col.flex-left/.flex-center/.flex-right` = the 3 zones.
- Desktop menus = `ul.header-nav.header-nav-main.nav`; mobile = `ul.mobile-nav` (shown via
  `.show-for-medium` / hidden via `.hide-for-medium`).
- Sticky header adds `.stuck` / elements with `.hide-for-sticky` disappear when stuck.

### Menu items & dropdowns (standard WP + Flatsome classes)
```
li.menu-item.menu-item-has-children
  ├─ a.nav-top-link  (+ i.icon-angle-down for the caret)
  └─ ul.sub-menu.nav-dropdown.nav-dropdown-default
       └─ li.menu-item.menu-item-type-{post_type|taxonomy|custom} > a
```
- Mega menu (custom menu layout) replaces the `ul.sub-menu` with a rendered UX Block instead.

### Footer — two valid forms
**A) Theme Options widget footer (default):**
```
footer#footer.footer-wrapper
  ├─ div.footer-widgets.footer.footer-1.dark
  │    └─ div.row > div.col > aside.widget ...
  └─ div.absolute-footer.dark            (copyright bar)
       └─ div.container > div.row > div.col   ← "Footer 1 / Footer 2" copyright text
```
**B) Custom block footer (block assigned to footer area):** renders like any UX Builder block —
`section.section.has-block > div.section-bg.fill + div.section-content.relative > div.row >
div.col > div.col-inner`, usually still followed by the `.absolute-footer` copyright bar.

### Sidebar (content + sidebar layout)
Enabled per area in Theme Options (Blog / Shop / Page). Content and sidebar are just grid
columns — typically 9 / 3:
```
div.row.row-large
  ├─ div.large-9.col            ← main content (article / products)
  └─ div.large-3.col            ← sidebar column
       └─ #sidebar.sidebar-inner > aside.widget ...        (pages / shop)
```
On **single posts** the sidebar column is `div.post-sidebar.large-3.col` and may use a sticky
wrapper: `div.is-sticky-column > div.is-sticky-column__inner > aside.widget`.
Full-width layouts simply omit the sidebar column and use `div.large-12.col`.

> Widget areas themselves are registered sidebars; output one inside the builder with
> `[ux_sidebar id="..."]` (see [shortcodes.md](shortcodes.md)).
